# YRust Chain P2P 网络教程

## 简介

YRust Chain 包含一个简化版的 P2P 网络实现，支持：
- TCP 节点间通信
- 区块广播
- 交易广播
- 节点发现

## 启动 P2P 节点

### 单节点模式

```bash
cargo run -- p2p
```

默认监听端口 8000

### 指定端口

```bash
cargo run -- p2p --port 8001
```

### 连接到其他节点

```bash
# 节点 1
cargo run -- p2p --port 8001

# 节点 2 (连接到节点1)
cargo run -- p2p --port 8002 --peer 127.0.0.1:8001
```

## P2P 命令

启动节点后，可以使用以下命令：

| 命令 | 说明 |
|------|------|
| `mine` | 挖一个新块并广播 |
| `status` | 查看区块链状态 |
| `peers` | 查看已连接节点 |
| `broadcast` | 发送测试消息 |
| `help` | 显示帮助 |
| `exit` | 退出节点 |

## 网络消息类型

```rust
pub enum NetworkMessage {
    Handshake { version, node_id },
    HandshakeAck { height, best_hash },
    NewBlock(Block),
    NewTransaction(Transaction),
    GetBlocks { start_height, end_height },
    BlocksResponse(Vec<Block>),
    Ping,
    Pong,
}
```

## 多节点网络

可以启动多个节点形成网络：

```bash
# 节点 1 (种子节点)
cargo run -- p2p --port 8001

# 节点 2
cargo run -- p2p --port 8002 --peer 127.0.0.1:8001

# 节点 3
cargo run -- p2p --port 8003 --peer 127.0.0.1:8001

# 节点 4 (连接到节点2)
cargo run -- p2p --port 8004 --peer 127.0.0.1:8002
```

## 注意事项

1. **当前是简化版**：使用 TCP 直接通信，没有使用 libp2p
2. **广播范围**：消息只发送给直接连接的节点
3. **数据同步**：需要手动实现完整的区块同步逻辑
4. **生产环境**：建议使用成熟的 P2P 库如 libp2p

## 扩展功能

要添加更多 P2P 功能，可以修改 `src/network/mod.rs`：

1. 添加自动发现机制
2. 实现完整的区块同步
3. 添加交易池同步
4. 使用 libp2p 替代简单 TCP
