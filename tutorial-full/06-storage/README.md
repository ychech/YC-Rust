# 06. 数据存储 - RocksDB与状态管理

## 本章目标

理解区块链数据存储，使用RocksDB实现高效持久化。

## 为什么需要专用存储？

```
区块链数据特点：
- 大量小键值对（交易、UTXO）
- 顺序写入（区块追加）
- 随机读取（查询交易）
- 不可修改（历史数据只读）

RocksDB优势：
- LSM树结构，写优化
- 列族隔离不同类型数据
- 压缩节省空间
- 快照备份
```

## 存储结构

```
Column Families:
├── blocks:      区块数据
├── transactions: 交易数据
├── utxos:       UTXO集合
├── state:       账户状态
└── metadata:    元数据（最新高度等）
```
