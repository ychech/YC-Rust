# 05. 共识算法 - Raft领导者选举

## 本章目标

理解分布式共识，实现Raft算法的领导者选举。

## 什么是共识？

共识是让分布式系统中的所有节点就某个值达成一致的过程。

```
问题：多个节点如何决定下一个区块？

解决方案：
1. 选出一个领导者（Leader）
2. 领导者负责出块
3. 其他节点（Follower）验证并跟随
4. 领导者失效时重新选举
```

## Raft核心概念

### 三种角色
```rust
pub enum NodeRole {
    Follower,   // 跟随者：接收并验证区块
    Candidate,  // 候选者：竞选领导者
    Leader,     // 领导者：负责出块
}
```

### 选举流程
```
1. 节点A超时未收到心跳，成为Candidate
2. 节点A增加term，向其他节点请求投票
3. 其他节点投票给A
4. A获得多数票，成为Leader
5. A开始发送心跳，阻止新选举
```
